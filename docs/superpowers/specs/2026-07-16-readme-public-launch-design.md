# README 与公开发布设计说明

**日期：** 2026-07-16  
**状态：** 视觉方向已由用户选择 A（产品展示型）  
**语言：** 中文主版 + 英文摘要

## 目标

把 `Codex Immersive Skin` 的 README 从功能说明书升级为可信、易读、具有视觉记忆点的开源项目首页，并在全部验证通过后将 GitHub 仓库从 Private 切换为 Public。

README 需要让第一次访问的用户在 30 秒内回答四个问题：

1. 这是什么；
2. 它能做什么；
3. 怎样安装和恢复；
4. 为什么可以信任它的来源与安全边界。

## 设计方向

采用“产品展示型”方案，参考 `travel-plan-viz` README 的视觉框架，而不是完整复制其内容体量：

- 使用居中 Hero、短价值主张、徽章和页内导航；
- 用表格和短段落建立清晰层级；
- 使用仓库已有的两张原创渐变背景作为真实主题预览；
- 不生成或伪造 Codex 界面截图；
- 不添加虚假的 Star 数、下载量、兼容平台或尚未实现的功能；
- 中文承担完整说明，英文只提供项目摘要、主要能力、安装入口和免责声明。

## 页面结构

### 1. Hero

居中显示：

- 项目名与一句价值主张；
- “macOS”“MIT”“Light / Dark”“Tests Passing”“Reversible”等状态徽章；
- 靠近标题的作者回流入口：`🎬 更多 AI 工具实战玩法：作者抖音 @泽轩604（App 内搜索即达）`；
- 特点、安装、自定义、安全、FAQ、English 等页内导航；
- 非 OpenAI 官方项目的短提示。

Hero 不使用带文字的 AI 图片，避免生成文字错误，也避免让装饰图抢过项目信息。

作者入口必须出现在 README 首屏，不埋在页尾。当前没有已确认可长期使用的抖音主页 URL，因此先使用清晰、可搜索的账号名和“App 内搜索即达”提示；不得把 `travel-plan-viz` 的项目视频短链误用到本仓库。

### 2. 30 秒理解

用四行功能表说明：

- 自定义图片成为沉浸式主题；
- 原生组件随浅色/深色主题适配；
- 刷新、路由切换或渲染器重建后自动重注入；
- 一键验证和恢复。

### 3. 主题预览

并排展示仓库中已有的真实资产：

- `assets/morning-mist.png`：浅色示例“晨雾”；
- `examples/warm-sand/background.png`：深色示例“暖沙”。

图片只描述背景风格，不声称它们是 Codex 界面截图。

### 4. 快速开始

保留两种安装方式，但优先突出零门槛路径：

1. 下载 ZIP；
2. 双击安装入口；
3. 从桌面定制入口选择自己的图片。

Git 克隆、CLI 定制、浅深色和强调色参数作为进阶内容保留。

### 5. 使用与恢复

解释桌面四个入口，并把“验证”和“彻底恢复”放在相邻位置。用户在安装前就能看到退出路径，降低对注入类工具的心理负担。

### 6. 工作原理与安全边界

保留现有技术事实：签名 Node.js、`127.0.0.1` CDP、进程身份核验、`app://` 页面限制、运行时重注入和恢复校验。

明确说明 CDP 的本机风险，不把“仅监听本机”夸张为“绝对安全”。

### 7. 来源、许可证与免责声明

上游署名在页面前部和许可证部分各出现一次：

- `HeiGeAi/codex-miku-theme` 与公众号“黑哥Ai”：创意参考。

保留 MIT、NOTICE、素材权利边界以及“非 OpenAI 发布、合作、授权或背书”的声明。

- README 的所有相对链接和图片路径可在 GitHub 正常解析；
- README 首屏出现作者抖音 `@泽轩604` 的回流入口；

- 上游归属、MIT、素材与非官方免责声明完整保留；
- 不出现未经验证的功能、数字或平台承诺；
- `./tests/run-tests.sh` 全部通过；
- 敏感信息扫描没有发现令牌、私钥、Cookie 或本机隐私路径；
- `git diff --check` 通过；
- 推送成功后再切换为 Public；
- 最终用 GitHub CLI 复查仓库可见性、默认分支和远端提交。

## 发布顺序

1. 在 Private 状态修改 README；
2. 本地测试、链接检查、敏感信息扫描和差异审阅；
3. 提交并推送到 `main`；
4. 再次确认远端提交；
5. 将仓库切换为 Public；
6. 复查公开状态和 README 页面。

公开是最后一步；任何验证失败都会阻止可见性切换。
