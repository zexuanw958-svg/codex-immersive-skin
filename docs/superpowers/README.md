# Historical workflow records

The dated specifications and implementation plans in this directory preserve the decisions and release constraints that applied when each change was designed. They are historical workflow evidence, not current operating instructions.

For the current supported platform, version, installation steps, release state, and known fixes, use the repository root `README.md`, `CHANGELOG.md`, `VERSION`, and the current Git history. Do not execute an old plan's unchecked steps or visibility commands without reconciling them against those current sources first.
