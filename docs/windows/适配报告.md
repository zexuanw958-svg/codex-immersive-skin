## 2. 结论摘要

Windows 五入口、Store/MSIX 身份与 Codex 自身运行时校验、本地图片归一化、自动/部分/全显式配色、事务安装、实时 Verify 和可重试 Restore 已完成实现。原始适配和 2026-07-19 启动/回滚补充复测都在本报告所列 Windows 11 真机上完成两轮 Start → Verify → Restore。

结论：**可以在报告所列的 Windows 11 x64、Microsoft Store `OpenAI.Codex` 26.715.2305.0 和 26.715.3651.0 精确环境范围内，先向小范围观众分阶段发布。** 发布前仍应运行 doctor；版本、架构、包形态或身份不一致时必须按“未验证”处理。现有结果不能外推为所有 Windows、Windows on ARM、Windows 10、旁加载或企业重打包均受支持。观众反馈所用 Windows 10 10.0.19045 与 Codex 26.715.4045.0 仍需原环境复测。

真机验证及补充复测期间发现并修复了五项用户可见问题：

1. 新建对话页的原生 composer 作为 flex item 时被 `margin-inline:auto` 压缩到约 26 px；移除该规则，并让 Verify 强制检查 composer、编辑区、滚动宽度和控件边界。
2. Windows PowerShell 5.1 会向无参数 `.cmd` 入口传入空的 remaining-argument 哨兵；五个生命周期入口现统一过滤空值。
3. 第二次安装校验无附加参数的快捷方式时，PowerShell 5.1 严格模式把空结果折叠为 `$null`，读取 `.Count` 失败；现用数组子表达式稳定保留空数组。
4. Start 假定 `Start-Process` 返回 PID 就是最终 Codex 主进程；Store/Electron 启动器短命或发生 PID 交接时会提前失败。现用事务基线、启动时间、随机标记、精确调试参数和新主进程重新发现完成交接，并保存全部已观察身份供失败回滚清理。
5. Windows PowerShell 5.1 把三参数 `File.Replace` 的 `$null` backup 绑定为非法空路径。现传入同目录唯一 displaced 路径，并在逐次 CAS 后只重试明确的 Windows 瞬时文件错误；目标与 displaced 哈希都通过后才清理。

上述修复均先复现失败，再通过针对性测试和真实安装副本复验。

## 4. 实现范围

### 4.1 Windows 用户入口

- `Install Codex Immersive Skin.cmd`
- `Customize Codex Immersive Skin.cmd`
- `Start Codex Immersive Skin.cmd`
- `Verify Codex Immersive Skin.cmd`
- `Restore Codex Immersive Skin.cmd`

所有 `.cmd` 入口只对新建 PowerShell 进程使用 `RemoteSigned`，不调用 `Set-ExecutionPolicy`，不要求关闭 SmartScreen，也不修改用户级或机器级策略。

### 4.2 安装与状态

- 默认安装到 `%USERPROFILE%\.codex\codex-immersive-skin`。
- 主题、状态、验证 JSON 和日志位于 `%LOCALAPPDATA%\CodexImmersiveSkin`。
- 安装器只替换带本项目身份标记的安装树，使用 `.installing`、`.previous`、`.failed` 和回滚目录完成事务部署。
- 桌面创建 Start、Customize、Verify、Restore 四个带身份说明的快捷方式；同名但不属于本项目的文件会触发失败并回滚。
- `--no-launch` 安装不读取、备份或写入 Codex 配置；两项基础配色只在 Start 已获得重启授权并关闭 Codex 后进入事务。

### 4.3 图片与配色

- 文件选择器、尺寸/字节限制、扩展名与真实解码检查均在本机完成。
- Windows 使用短生命周期 `System.Drawing` helper 归一化为 JPEG；最长边不超过 3200 px，输出不超过 16 MB。
- 支持全自动、只覆盖部分参数和全显式参数；未显式项继续从同一张归一化图片分析。
- 分析失败使用安全默认配色并警告；转换失败保持原主题字节不变。

### 4.4 生命周期与安全边界

- 只接受当前用户注册、身份固定且通过 BlockMap/签名检查的 Store/MSIX 包。
- 运行时使用 Codex 自己准备的 `%LOCALAPPDATA%` Node，哈希必须匹配包内副本；不复制 WindowsApps 文件，不依赖全局 Node。
- CDP 仅绑定 `127.0.0.1`；端点、监听进程、包身份、进程启动时间和命令行均参与核验。
- Start/Restore/Install/Customize/Verify 使用按当前用户 SID 和状态根派生的 `Global` 命名互斥锁，ACL 仅允许当前用户。
- 温和关闭和必要的终止都在同一已验证进程句柄上复核身份，避免按进程名批量停止。
- 配置、主题目录、安装树与快捷方式使用 CAS、所有权标记和逐级 reparse 检查。
- Restore 先只读验证恢复材料，保留备份直到实时 DOM、配置、正常启动和状态清理都提交完成。
- 不修改 MSIX、`app.asar`、应用签名或注册表执行策略。

### 6.1 配置完整性说明

首次安装前采样的整文件为 4,285 bytes，SHA-256 `30BA2C58FB0DF421B2C1B587D70A21DBC86E606BF8BE94CA980AD7F1BF61EC19`。真实生命周期内，Codex 会在新建任务、模型/会话状态变化和正常重启时写回同一份配置，因此 Restore 后整文件哈希没有保持为最初采样值。

本项目的恢复语义不是用旧快照覆盖整份配置，而是：只保存并恢复 `appearanceTheme` 与 `appearanceDarkCodeThemeId` 两个基础主题键；写入前校验整文件 CAS；检测到并发编辑时停止覆盖；保留其他 Codex 或用户设置。两轮 Restore 都在 Codex 关闭后完成选择性恢复，随后才删除备份并正常启动 Codex。根据写入时点和选择性恢复实现判断，整文件哈希变化来自生命周期中的非主题设置写回；本报告不把“整文件哈希不等”误报为主题恢复失败，也不声称覆盖了用户的并发设置。

## 8. 实测避坑与最小解法

1. **上游测试直接调用 `/bin/bash`。** 使用跨平台 Node 分派器；Windows 进入 PowerShell runner，macOS 原 runner 保持不变。
2. **图片工具写死 `/usr/bin/sips`。** 保持分析算法和 CLI 不变，只替换 Windows 归一化后端为短生命周期 `System.Drawing` helper。
3. **WindowsApps 内 Node 不适合直接作为普通文件执行。** 使用 Codex 首次启动后准备的本地运行时，并要求其哈希匹配包内副本。
4. **仅验证“签名有效”不能证明官方 Store 包身份。** 同时固定包名、Publisher/Publisher ID、SignatureKind、入口和 BlockMap 内容链。
5. **普通后代进程可伪造端口。** 监听者本身必须是已验证的 `ChatGPT.exe`，并继续核对 loopback `/json/version` 与 renderer 结构。
6. **PowerShell 5.1 对无 BOM 中文脚本不稳定。** 含中文的 `.ps1` 使用 UTF-8 BOM；Node 子进程 stdout/stderr 显式按严格 UTF-8 解码。
7. **字符串前缀不能阻止 junction 越界。** 对可变目录逐级拒绝 reparse，并在递归移动/删除前再次校验整棵树。
8. **安装失败时无条件回滚快捷方式可能误删同名私人文件。** 只有完成身份检查和事务备份后才启用快捷方式回滚，并用写入哈希 CAS。
9. **Start 在事务外关闭 Codex 会留下中断状态。** 把关闭、配置快照、启动、注入器和状态写入纳入同一可恢复事务。
10. **Restore 过早删除备份会失去重试能力。** 先只读验证，恢复时 `--keep-backup`，所有阶段成功后再删除。
11. **并发配置编辑不能用旧整文件快照覆盖。** 只恢复两项主题键，所有写入和回滚都要求预期哈希相符。
12. **默认截图可能泄露私人任务。** Windows Verify 默认只保存 JSON；截图必须针对非敏感任务另行取得明确同意。
13. **flex item 的 `margin-inline:auto` 会触发 min-content 收缩。** 不给原生 composer 自动外边距；Verify 至少要求 240 px 宽、编辑区可见、滚动宽度不越界、控件边界在容器内。
14. **只验证“外壳存在”会漏掉内部布局塌缩。** Verify 现同时采集 composer、编辑区、client/scroll width 和控件边界，并把 `composerLayoutOk` 纳入最终 `pass`。
15. **无参数 `.cmd` 在 PowerShell 5.1 可产生空参数哨兵。** 五个入口在解析前统一过滤 null/empty 值。
16. **PowerShell 管道会把空数组折叠为 `$null`。** 快捷方式附加参数使用外层 `@(...)` 数组子表达式，第二次安装不再访问不存在的 `.Count`。
17. **测试窗体设置 `ShowInTaskbar=false` 会变成 owned window。** 温和关闭测试使用可见、无 owner 的主窗口，与生产身份约束一致。
18. **Windows 防护软件可短暂拒绝原子重命名。** 仅对带预期哈希的替换在每次重试前重新执行 CAS，针对 `EPERM`/`EACCES`/`EBUSY` 做 8 次有界退避；其他错误立即失败。

## 10. 当前限制与未验证事项

- Windows 11 x64 之外的平台未验证。
- 只支持通过固定身份检查的 Microsoft Store/MSIX `OpenAI.Codex`；旁加载、企业重打包和其他发行渠道未验证。
- 本报告验证 Codex 26.715.2305.0，并在启动/回滚补充复测中验证 26.715.3651.0；其他升级版本可能改变包、运行时或 DOM，doctor/Verify 失败时应停止并重新适配。
- CDP 为 loopback-only 但无额外认证，不应在主题会话期间运行不可信本地程序。
- 自动 Verify 和一次人工确认不能穷尽所有任务、缩放比例、多显示器、辅助功能和语言组合。
- 配置/快捷方式 CAS 可阻止普通并发编辑被覆盖，但不是抵御同一用户恶意进程的安全隔离；最终提交前仍存在极窄 TOCTOU 窗口。
- 为兼容早期安装，`--reset-demo` 对精确受管主题目录保留 legacy schema 认领；任意目录、测试边界外目录和结构不匹配目录均拒绝。
- `--uninstall` 只删除身份匹配的桌面快捷方式，不删除安装目录和用户主题。

## 11. 发布建议

发布建议为：**可以在已验证环境内小范围、分阶段发布；暂不应宣传为所有 Windows 通用。**

面向观众时必须同时提供：

1. 先正常启动一次 Store Codex，再运行 `doctor-windows.ps1`；只有 `pass=true` 且版本/架构与本报告一致时继续。
2. 不关闭 SmartScreen，不持久修改执行策略；只对来源可信且核过哈希的 ZIP 解除锁定。
3. Start 会关闭并重启 Codex；运行前保存未完成输入。
4. Verify 自动通过后仍要实际点击侧栏、输入框、按钮和链接。
5. 不用主题时运行 Restore，并确认 doctor 显示 `statePresent=false`、`cdpVerified=false`、`live=false`。
6. 遇到不一致、失败或升级后的未知 DOM，不绕过身份/布局校验，回到 `windows-support` 分支重新适配。

## 13. 2026-07-19 启动失败与配置回滚补充复测

### 13.1 反馈与根因边界

观众环境为 Windows 10 10.0.19045、Windows PowerShell 5.1、Store Codex `26.715.4045.0`。反馈同时包含“调试主进程身份未建立”和 `File.Replace` 路径格式异常。两条失败分别定位：前者来自脚本把短生命周期启动句柄当作最终主进程；后者来自 PowerShell 5.1 对 `$null` backup 路径的重载绑定。二者发生在同一失败事务中，但属于不同代码缺陷。

本机为 Windows 11 家庭中文版 10.0.26200.8655 x64、Windows PowerShell 5.1.26100.8655、Store Codex `26.715.3651.0`、Codex 自身 Node 24.14.0 x64。当前机器证明通用修复在该精确组合通过，不能证明 Windows 10 或 `26.715.4045.0` 已通过。

### 13.2 修复设计与安全边界

- Start 记录启动前无主进程基线、UTC 时间和 32 位十六进制事务标记；只接纳唯一、启动后出现、包身份通过并精确携带 `127.0.0.1`、选定端口和事务标记的主进程。

- 配置快照恢复给 `File.Replace` 传入同目录唯一 displaced 路径。每次尝试前复核当前配置哈希；只对 access denied、sharing violation、lock violation 做 8 次以内退避；恢复后核对目标与 displaced 双哈希。
